相册重构+偷懒史

本站Photos页面重构。全部相片托管在本地。

本站的Photos页面进化过程就是本人的一部偷懒史。但如懒人图库的Slogen：要懒出境界。

—————————————————————————————–

V1

最早的相片展示页，是静态的html，每一个标签都是手动地复制粘贴。稍微自动化一点的地方就是图片的大小调整通过Photoshop的“动作”来完成。这样的操作足以让我在第一次初始构建页面后再也不添加新相片，真正让它“静态”了。

—————————————————————————————–

V2

学过了动态网页技术并完成学院网站建设后，我把学院网站后台的部分模块移植到我的个人站。于是通过Access+asp，相片这块“动”起来了。后台只能单张上传，每张需要上传一张大图一张缩略图、选择类别、填标题、写简介。

上传前十张的时候，我还兴致高昂地全填；

上传第二个十张时，省了简介。

第三个十张时，只选类别。

又传了两张，我撂手了。不胜其烦。于是，很长的一段时间，Photos只有32张图。

但在前台的读取上，花了点小心思：我写了一个Javascript，将乱序的32张图随机地散列到10*15的方格中。如此，每次刷新都重新排列，看客也就觉得好像还有蛮多的图或者好新鲜。

点击查看效果

—————————————————————————————–

V3

后来成了Google迷。用上了Picasa Web Album。之后这也成为了我爱上Google有一个理由。通过Google的Picasa桌面软件——恰好我本来就用它管理相片。可以一键上传。这个一键上传可包含了很多内容：Picasa负责把你的相片调整成适合Web显示的大小、加上Logo的水印、缩放裁切的缩略图等。然后Picasa Web Album提供了免费的2G大小空间，Google提供了开放数据的API。可以通过API，将图片调用到本地。

好处多多：

  1. 方便。点一下就全部完成。
  2. 优质有效。借助API和AJAX技术，整个相册的实体只有一个页面，几十K的大小。相册列表、缩略图显示、大图显示均在一个页面内完成，节省空间。
  3. 稳定。Google的服务器诶，世界上还有比其更稳定的么？没有了。

悲剧的是，不久满页的红叉叉告诉我一个严重的忽视：我忘记了考虑中国著名的长城。

—————————————————————————————–

V4

Picasa Web Album的被墙后，我开始使用XML+XTSL构建Photos。这是因为一次偶然发现通过Picasa，一组相片可以输出大图+缩略图+包含指向上述二者的XML！我每次更新相册要做的就是选个文件夹“导出为……”一下，然后在XML头部粘贴一行XTSL的关联代码。

这个推荐给所有有独立相册程序，特别是Flash+XML的童鞋使用。

这个XML的模板也是可以自定义的。模板路径在：

Windows：程序文件夹/Web/Templates/xml/

Mac OS X：包内容/Resources/web/templates/xml/

—————————————————————————————–

V5

现在的版本。我觉得应该是独立相册的最高偷懒境界了。

更新图片要做的，只要上传就可以了。

通过php自动读取FTP上该目录下所有图。

然后我用Mac的小机器人来做缩略图的生成和重命名。

爽啊，以后每次把相片从相机取出时顺手上传一份。啥都不用改。灭哈哈哈哈。
